## 优惠券模块API接口文档

### 一、概述

本文档描述了悦享生活平台优惠券模块的所有API接口，包括C端用户接口和B端商户接口。

**基础URL**: `http://localhost:8080`

**通用请求头**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Content-Type | String | 是 | application/json |
| Authorization | String | 部分 | Bearer {token}，需要认证的接口必填 |

**通用响应格式**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

##### 2.1.1 获取秒杀券列表

**接口名称**: 获取秒杀券列表  
**请求URL**: `/api/seckill/list`

**认证要求**: 无需认证

**请求参数**:
| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| pageSize | Integer | 否 | 6 | 每页数量，上限10 |

**成功响应示例**:
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "serverTime": "2025-03-28T10:30:00",
    "records": [
      {
        "voucherId": 501,
        "shopId": 1001,

        "endTime": "2025-03-28T18:00:00",
        "remainSeconds": 27000,
        "status": 1,
        "image": "shop/2025/chuanwei.jpg"
      }
    ]
  }
}
```

**响应字段说明**:
| 字段名 | 类型 | 说明 |
|--------|------|------|
| serverTime | String | 服务器当前时间 |
| voucherId | Long | 优惠券ID |
| shopId | Long | 商户ID |
| shopName | String | 商户名称 |
| title | String | 券标题 |
| payValue | Double | 秒杀价（元） |
| actualValue | Double | 抵扣面值（元） |
| stock | Integer | 剩余库存 |
| totalStock | Integer | 初始总库存 |
| beginTime | String | 秒杀开始时间 |
| endTime | String | 秒杀结束时间 |
| remainSeconds | Long | 距结束剩余秒数 |
| status | Integer | 状态：0未开始，1进行中，2已结束，3已抢光 |
| image | String | 券封面图 |

##### 2.1.3 秒杀下单

**接口名称**: 秒杀下单  
**请求URL**: `/api/seckill/order/{id}`  
**请求方法**: `POST`  
**认证要求**: 必须登录

**请求头**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| Authorization | String | 是 | Bearer {token} |

**路径参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 优惠券ID |

**成功响应示例**:
```json
{
  "code": 200,
  "msg": "success",
  "data": 1234567890123456
}
```

**说明**: `data` 返回订单ID，用户需轮询查询订单结果。

**错误响应**:
| code | msg | 说明 |
|------|-----|------|
| 500 | 库存不足 | 商品已售罄 |
| 500 | 每人限购一张 | 用户已购买过 |
| 500 | 秒杀活动不存在 | 活动未开始或已结束 |
| 500 | 请勿重复下单 | 重复请求 |

### 四、错误码说明

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 401 | 未登录 |
| 403 | 无权限 |
| 400 | 参数错误 |
| 500 | 业务异常 |

**常见业务异常**:
| msg | 说明 |
|-----|------|
| 库存不足 | 秒杀商品已售罄 |
| 每人限购一张 | 用户已购买过该商品 |
| 秒杀活动不存在 | 活动未开始或已结束 |
| 请勿重复下单 | 重复请求 |
| 订单不存在 | 订单ID无效 |
| 订单状态异常 | 订单当前状态不允许此操作 |
| 只能取消未支付的订单 | 订单已支付无法取消 |
| 只能退款已支付的订单 | 订单未支付或已使用 |
| 上架中的优惠券不能删除 | 需先下架 |

### 五、调用注意事项

1. **秒杀下单**: 返回订单ID后需轮询查询订单结果，建议间隔500ms
2. **库存同步**: 秒杀库存以Redis为准，MySQL库存为备份
3. **一人一单**: 同一用户对同一秒杀券只能购买一次
4. **订单超时**: 未支付订单建议设置自动取消时间（如15分钟）
5. **并发控制**: 秒杀接口使用Redis Lua脚本保证原子性
6. **缓存策略**: 秒杀列表缓存5分钟，库存实时从Redis读取
