## 优惠券使用流程

### 一、用户端使用流程

#### 1. 获取优惠券

**方式一：购买普通优惠券**
```
1. 浏览商户优惠券列表 → GET /api/voucher/shop/{shopId}
2. 查看优惠券详情 → GET /api/voucher/{id}
3. 购买优惠券（需调用支付接口）
```

**方式二：参与秒杀抢购**
```
1. 查看秒杀列表 → GET /api/seckill/list
2. 查看秒杀详情 → GET /api/seckill/{id}
3. 秒杀下单 → POST /api/seckill/order/{id}
4. 轮询订单结果 → GET /api/seckill/result/{orderId}
5. 支付订单 → POST /api/voucher-order/{id}/pay
```

#### 2. 查看我的优惠券

```
GET /api/voucher/my?status=1
```
- `status=1`：已支付，可用于消费
- `status=2`：已使用
- `status=3`：已退款

#### 3. 到店使用

用户到商户门店消费时：
1. 打开APP展示优惠券二维码或订单编号
2. 商户扫码或输入订单编号核销
3. 核销成功后，优惠券状态变为"已使用"

---

### 二、商户端核销流程

#### 1. 查看订单

```
GET /api/shop/voucher/order?status=1
```
查看已支付待核销的订单列表

#### 2. 核销优惠券

```
POST /api/shop/voucher/order/verify?orderNo=YX20250328123456ABCD
```

**请求头**:
```
X-Shop-Id: 1001
```

**成功响应**:
```json
{
  "code": 200,
  "msg": "success",
  "data": true
}
```

### 四、订单状态流转

| 状态码 | 状态名 | 说明 | 可执行操作 |
|--------|--------|------|------------|
| 0 | 未支付 | 订单已创建，等待支付 | 支付、取消 |
| 1 | 已支付 | 已完成支付，可到店使用 | 核销、退款 |
| 2 | 已使用 | 已在商户处消费完成 | 无 |
| 3 | 已退款 | 已申请退款 | 无 |
| 4 | 已取消 | 用户主动取消 | 无 |

**状态流转图**:
```
未支付(0) ──支付──→ 已支付(1) ──核销──→ 已使用(2)
    │                  │
    │取消              │退款
    ↓                  ↓
已取消(4)           已退款(3)
```

### 五、使用示例

#### 场景：用户购买秒杀券并到店使用

**Step 1: 用户查看秒杀列表**
```bash
curl -X GET "http://localhost:8080/api/seckill/list?pageSize=6"
```

**Step 2: 用户秒杀下单**
```bash
curl -X POST "http://localhost:8080/api/seckill/order/501" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

返回订单ID: `1234567890123456`

**Step 3: 用户支付订单**
```bash
curl -X POST "http://localhost:8080/api/voucher-order/1234567890123456/pay?payType=2" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

**Step 4: 用户到店出示订单编号**

用户在APP中查看订单详情，展示订单编号：`YX20250328123456ABCD`

**Step 5: 商户核销**
```bash
curl -X POST "http://localhost:8080/api/shop/voucher/order/verify?orderNo=YX20250328123456ABCD" \
  -H "X-Shop-Id: 1001" \
  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
```

核销成功后，订单状态变为"已使用(2)"。

### 六、注意事项

1. **有效期**: 优惠券有使用有效期，过期无法使用
2. **一人一单**: 秒杀券每人限购一张
3. **退款规则**: 已使用的优惠券无法退款
4. **核销权限**: 只有券所属商户才能核销
5. **订单超时**: 未支付订单建议15分钟内完成支付
