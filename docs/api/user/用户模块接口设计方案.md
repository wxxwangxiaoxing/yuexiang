# 用户模块接口设计方案（修订版 v2.0）

---

## 一、模块概述

用户模块负责用户信息管理、账户安全、密码管理等功能。认证登录功能已独立到系统模块（认证模块）。

### 模块职责划分

| 模块 | 职责 | 说明 |
|------|------|------|
| **系统模块（认证）** | 登录注册、Token管理、会话管理 | 系统级基础设施 |
| **用户模块** | 用户信息管理、密码管理、账户安全 | 用户业务功能 |

> 所有需要验证码的场景统一调用 `POST /api/auth/sms-code`，通过 `type` 参数区分用途。每种 type 的验证码互不通用。

| type | 用途 | 调用场景 |
|:----:|------|---------|

| 1 | 用户注册 | 注册页 |
| 2 | 找回/重置密码 | 忘记密码页 |
| 3 | 绑定新手机号 | 修改手机号（新号码） |
| 4 | 验证原手机号 | 修改手机号（旧号码） |
| 5 | 设置登录密码 | 设置密码页 |
| 6 | 设置/重置支付密码 | 支付密码页 |
| 7 | 注销账户 | 注销申请 / 撤销注销 |

## 六、密码强度规则

### 6.1 登录密码

| 规则 | 说明 |
|------|------|
| 长度 | 8~20 位 |
| 字符要求 | 至少包含字母 + 数字 |
| 禁止 | 纯连续数字（12345678） |
| 禁止 | 纯重复字符（aaaaaaaa） |
| 禁止 | 与手机号后 8 位相同 |

### 6.2 支付密码

| 规则 | 说明 |
|------|------|
| 长度 | 固定 6 位数字 |
| 禁止 | 连续数字（123456 / 654321） |
| 禁止 | 重复数字（111111 / 888888） |
| 禁止 | 与手机号后 6 位相同 |

## 七、支付密码安全策略

```plain
存储: Redis INCR  pay_pwd_err:{userId}  TTL=24h
──────────────────────────────────────────────────

  连续错误 ≥ 3 次 → 锁定 2 小时
    Redis SET pay_pwd_lock:{userId} "1" EX 7200

  连续错误 ≥ 5 次 → 锁定 24 小时
    Redis SET pay_pwd_lock:{userId} "1" EX 86400

  连续错误 ≥ 10 次 → 永久锁定
    Redis SET pay_pwd_lock:{userId} "1"  (无过期)
    需通过手机验证码重置支付密码后解锁

  成功验证 → 重置计数器
    Redis DEL pay_pwd_err:{userId}
    Redis DEL pay_pwd_lock:{userId}

  解锁方式:
    自动解锁: 锁定到期后自动恢复（1/2级）
    手动解锁: 手机验证码验证 → 重置支付密码（3级）
```

## 八、密码变更后 Token 失效策略

```plain
适用场景: 修改密码 / 重置密码
──────────────────────────────────────────────────

① 服务端处理:
   · 修改密码成功后，将用户所有活跃 Token 的 jti 加入 Redis 黑名单
     Key:  token:blacklist:{jti}
     TTL:  该 Token 剩余有效时间

   · 同时更新 tb_user.update_time
     后续 Token 校验中间件可对比 iat < update_time → 拒绝

② 响应中携带标记:
   { "code": 200, "msg": "修改成功", "data": { "requireReLogin": true } }

③ 前端处理:
   收到 requireReLogin=true → 清除本地 Token → 跳转登录页

④ 踢出所有设备:
   如果用户有多设备登录（设备管理功能），密码变更后
   所有设备的 Token 全部失效，需重新登录
```

### 10.9 修改支付密码

#### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| oldPayPassword | String | 是 | 原支付密码 |

| confirmPassword | String | 是 | 确认新密码 |

```json
}
```

```json
{
  "code": 200,
  "msg": "修改成功",
  "data": null
}
```

#### 错误响应

| code | msg | 场景 |
|:----:|-----|------|
| 40034 | 原支付密码错误 | BCrypt 比对失败 |

| 40304 | 支付密码已永久锁定，请通过验证码重置 | 连续错误 ≥ 10 次 |

```plain
   └─ 成功:

       DEL pay_pwd_lock:{userId}

③ 校验 newPayPassword 安全规则 + confirmPassword 一致
④ 更新 tb_user_wallet.pay_password = BCrypt(newPayPassword)
⑤ 记录操作日志
```

#### 自动注销定时任务

```plain

SELECT id, phone FROM tb_user
WHERE status = 3
  AND cancel_deadline < NOW()
  AND deleted = 0;

对每个用户执行:
① tb_user.deleted = 1, status = 0
② tb_user_info.deleted = 1
③ tb_user_wallet.deleted = 1
④ tb_user_auth.deleted = 1
⑤ 清除 Redis 中所有 Token（黑名单机制）
⑥ 记录操作日志
⑦ (可选) 异步清理关联内容（笔记/评论匿名化等）
```

### 10.14 撤销注销申请

**POST** `/api/user/cancel/revoke`

#### 功能描述

在冷静期内撤销注销申请。需要手机验证码验证（防止 Token 被盗后恶意操作）。

#### 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| code | String | 是 | 手机验证码(type=7) |

#### 请求示例

```json
{
  "code": "123456"
}
```

#### 成功响应

```json
{
  "code": 200,
  "msg": "撤销成功，账户已恢复正常",
  "data": null
}
```

#### 错误响应

| code | msg | 场景 |
|:----:|-----|------|
| 40053 | 无注销申请 | status != 3 |
| 40054 | 冷静期已过，无法撤销 | cancel_deadline < NOW() |
| 40002 | 验证码错误 | code 不匹配 |

#### 后端处理流程

```plain
① 校验验证码（type=7）
② 检查 tb_user.status == 3 → 否则 40053
③ 检查 cancel_deadline > NOW() → 否则 40054
④ 更新 tb_user:
   status = 0 (正常)
   cancel_time = NULL
   cancel_deadline = NULL
   cancel_reason = NULL
⑤ 记录操作日志
```

## 十一、安全措施汇总

| 安全措施 | 说明 | 实现方式 |
|----------|------|---------|
| HTTPS 强制 | 所有接口必须 HTTPS | Nginx 配置 |
| HSTS 头 | 强制 HTTPS，有效期 1 年 | 响应头 |
| 安全响应头 | X-Content-Type-Options, X-Frame-Options | 响应头 |
| 登录密码加密 | BCrypt 加密，强度 10 | Spring Security |
| 支付密码加密 | BCrypt 加密，强度 10 | 独立校验逻辑 |
| 手机号脱敏 | 中间 4 位打码 | VO 层处理 |
| 身份证加密 | AES-256 加密存储 | 服务层加解密 |
| 身份证脱敏 | 返回时仅显示前4+后2 | VO 层处理 |
| 姓名脱敏 | 仅显示姓氏 + * | VO 层处理 |
| 支付密码梯度锁定 | 3/5/10 次分级锁定 | Redis 计数器 |
| 密码变更踢出 | 所有 Token 加黑名单 | Redis 黑名单 |
| 敏感操作二次验证 | 改手机号/设密码/注销均需验证码 | 短信验证码 |
| 审计日志 | 敏感操作记录 180 天 | tb_operation_log |
| 接口限流 | 修改类接口限频 | Redis 滑动窗口 |
