# 商户列表页 — 后端接口设计

------

```plain
本页共需 2 个接口，核心为商户列表查询
```

| ---- | ------ | --------------------- | -------------------- |
| 1    | `GET`  | `/api/shop/list`      | 商户列表查询（核心） |

## 二、接口详细设计

### 接口 1：商户列表查询

```plain
GET /api/shop/list
```

#### 请求参数（Query）

```plain
所有参数均为可选，支持自由组合
```

| 参数        | 类型    | 必填 | 默认值  | 说明                               |
| ----------- | ------- | ---- | ------- | ---------------------------------- |
| `typeId`    | Long    | 否   | —       | 商户类型 ID                        |
| `keyword`   | String  | 否   | —       | 搜索关键词（匹配商户名称）         |
| `area`      | String  | 否   | —       | 商圈区域（如"望京"）               |
| `sortBy`    | String  | 否   | `score` | 排序方式 ↓                         |
| `longitude` | Double  | 条件 | —       | 用户经度（sortBy=distance 时必传） |
| `latitude`  | Double  | 条件 | —       | 用户纬度（sortBy=distance 时必传） |
| `minPrice`  | Integer | 否   | —       | 最低人均（元）                     |
| `maxPrice`  | Integer | 否   | —       | 最高人均（元）                     |
| `pageNo`    | Integer | 否   | 1       | 页码                               |
| `pageSize`  | Integer | 否   | 10      | 每页条数（上限 20）                |

**sortBy 可选值：**

| 值           | 说明         | 排序逻辑                   |
| ------------ | ------------ | -------------------------- |
| `distance`   | 距离优先     | 按距离升序（需经纬度）     |
| `score`      | 好评优先     | 按综合评分降序             |
| `price_asc`  | 人均从低到高 | 按 avg_price 升序          |
| `price_desc` | 人均从高到低 | 按 avg_price 降序          |
| `ai`         | AI 智能推荐  | 综合评分 × 距离 × 偏好加权 |

```http
GET /api/shop/list?typeId=1&area=望京&sortBy=distance

    &pageNo=1&pageSize=10

Authorization: Bearer <token>     ← 可选（登录后可获取收藏状态）
```

## 三、核心实现逻辑

### 3.1 Service 处理流程

```plain
ShopListService.queryShopList(ShopListQuery query)
│
├── 1. 参数校验
│   ├── sortBy=distance → 校验经纬度非空
│   ├── pageSize 上限 20
│   └── keyword 长度 ≤ 50
│
├── 2. 构建查询条件
│   ├── WHERE deleted = 0
│   ├── AND type_id = ?          ← typeId 非空时
│   ├── AND area = ?             ← area 非空时
│   ├── AND name LIKE %?%        ← keyword 非空时
│   └── AND avg_price BETWEEN    ← 价格区间过滤
│       ? * 100 AND ? * 100        (元 → 分)
│
├── 3. 排序策略
│   ├── distance → 计算距离并排序
│   ├── score    → ORDER BY score DESC
│   ├── price_*  → ORDER BY avg_price ASC/DESC
│   └── ai       → ORDER BY ai_score DESC
│
├── 4. 分页查询 tb_shop
│
├── 5. 批量补充关联数据
│   ├── 标签: IN 查询 tb_tag_relation + tb_tag
│   ├── 收藏状态: IN 查询 tb_favorite (登录时)
│   └── AI 摘要: 读取缓存或生成
│
└── 6. 组装 VO 返回
```

### 3.2 距离计算

```sql
-- 方案一：MySQL 球面距离函数
SELECT s.*,
       ST_Distance_Sphere(
           POINT(s.longitude, s.latitude),
           POINT(#{longitude}, #{latitude})
       ) AS distance
  FROM tb_shop s
 WHERE s.deleted = 0
   AND s.type_id = #{typeId}
 ORDER BY distance ASC
 LIMIT #{offset}, #{pageSize};
-- 方案二：先用矩形边界框粗筛，再精算（性能更优）
-- 1° 纬度 ≈ 111km，先过滤 5km 范围内

SET @lat = #{latitude};
SET @lng = #{longitude};
SET @radius = 5;  -- km

SELECT s.*,
       ST_Distance_Sphere(
           POINT(s.longitude, s.latitude),
           POINT(@lng, @lat)
       ) AS distance
  FROM tb_shop s
 WHERE s.deleted = 0
   AND s.type_id = #{typeId}
   AND s.latitude  BETWEEN @lat - @radius/111
                        AND @lat + @radius/111
   AND s.longitude BETWEEN @lng - @radius/(111 * COS(RADIANS(@lat)))
                        AND @lng + @radius/(111 * COS(RADIANS(@lat)))
 ORDER BY distance ASC
 LIMIT #{offset}, #{pageSize};
推荐：生产环境使用 Redis GEO 或 Elasticsearch geo_distance
      MySQL 方案适用于中小数据量（< 10 万商户）
```

### 3.3 AI 推荐排序

```plain
sortBy = ai 时的加权评分公式：

ai_score = w1 × normalized_score        -- 评分权重（0.35）
         + w2 × normalized_popularity   -- 热度权重（0.25）
         + w3 × normalized_distance     -- 距离权重（0.20）
         + w4 × user_preference_score   -- 用户偏好（0.20）

其中：
  normalized_score      = score / 5.0
  normalized_popularity = LOG(sales_count + 1) / MAX_LOG
  normalized_distance   = 1 - (distance / max_distance)
  user_preference_score = 基于浏览历史 + 收藏 + 消费记录的协同过滤得分
实现策略：
  ├── 轻量版：预计算 ai_score 存入 tb_shop 字段，定时任务更新
  └── 完整版：接入推荐引擎，实时个性化排序
```

### 3.4 标签批量查询

```sql
-- 根据商户 ID 列表批量查询标签
SELECT tr.biz_id AS shop_id,
       t.name    AS tag_name
  FROM tb_tag_relation tr
  JOIN tb_tag t ON t.id = tr.tag_id AND t.deleted = 0
 WHERE tr.biz_type = 0              -- 0 = 商户标签
   AND tr.biz_id IN (#{shopIds})    -- 当前页商户 ID 列表
 ORDER BY t.sort ASC;
```

### 3.5 收藏状态批量查询

```sql
-- 登录用户才执行
SELECT biz_id
  FROM tb_favorite
 WHERE user_id  = #{currentUserId}
   AND biz_type = 1                  -- 1 = 商户
   AND biz_id  IN (#{shopIds})
   AND deleted  = 0;
```

## 四、缓存策略

```plain
┌─────────────────────┬──────────────┬──────────┬───────────────────────┐
│ 缓存 Key             │ 数据来源      │ 过期时间  │ 更新策略               │
├─────────────────────┼──────────────┼──────────┼───────────────────────┤
│ shop:type:{typeId}  │ tb_shop_type │ 24h      │ 修改类型时删除          │
│ shop:area:list      │ tb_shop      │ 1h       │ 新增/修改商户时删除      │
│ shop:detail:{id}    │ tb_shop      │ 30min    │ 修改时删除 + 延迟双删   │
│ shop:geo:{typeId}   │ Redis GEO    │ 不过期    │ 商户坐标变更时 GEOADD   │
│ shop:tags:{id}      │ tag_relation │ 2h       │ 标签变更时删除          │
│ shop:ai:summary:{id}│ AI 生成       │ 7d       │ 新增评价时异步刷新      │
└─────────────────────┴──────────────┴──────────┴───────────────────────┘
热点商户（访问量 Top 100）：缓存预热 + 永不过期 + 主动更新
```
