# 优惠券使用后用户模块更新计划（修订版）

## 一、背景

当用户购买/使用优惠券后，需要更新用户模块的相关数据，包括：
- 用户钱包余额（支付扣款/退款返还）
- 用户积分（消费赠送积分）
- 会员等级（积分达标自动变更）

### 3.2 模块依赖关系

```
                 ┌─────────────────────┐
                 │   yuexiang-common    │
                 │   ├── MoneyUtils     │
                 │   └── 公共异常/结果   │
                 └──────────▲──────────┘
                            │
                 ┌──────────┴──────────┐
                 │   user-api           │
                 │   ├── 接口定义        │
                 │   ├── entity/vo      │
                 │   └── constants      │
                 └──────────▲──────────┘
                    ┌───────┴───────┐
                    │               │
          ┌─────────┴───┐   ┌──────┴──────────┐
          │  user-biz    │   │  voucher-biz     │
          │  接口实现     │   │  注入接口调用     │
          │  mapper      │   │                  │
          └──────┬──────┘   └─────────────────┘
                 │
          ┌──────┴──────┐
          │  server      │
          │  (启动模块)   │
          └─────────────┘

关键：
  - user-api   放接口 + 实体 + 常量
  - user-biz   放实现 + mapper + controller
  - voucher-biz 只依赖 user-api（面向接口编程）
  - server 模块必须依赖 user-biz，确保实现类被 Spring 扫描到
  - 运行时由 Spring 自动注入 user-biz 中的实现类
```

**yuexiang-server 启动类（确保扫描路径覆盖新模块）**：
```java
@SpringBootApplication
@MapperScan({
    "com.yuexiang.user.mapper",       // ← 新增
    "com.yuexiang.voucher.mapper",
    "com.yuexiang.shop.mapper",
    "com.yuexiang.system.mapper",
    "com.yuexiang.blog.mapper",
    "com.yuexiang.social.mapper"
})
public class YuexiangServerApplication {
    public static void main(String[] args) {
        SpringApplication.run(YuexiangServerApplication.class, args);
    }
}
```

> 如果启动类使用了 `@ComponentScan("com.yuexiang")`，则 Service 类无需额外配置。若指定了具体包路径，需确保 `com.yuexiang.user` 被包含。

### 3.4 涉及的数据库表

| 表名 | 说明 | 更新场景 |
|------|------|----------|
| `tb_user` | 用户基本信息表 | 查询用户昵称等 |
| `tb_user_info` | 用户详情表 | 积分累计、等级变更 |
| `tb_user_wallet` | 用户钱包表 | 支付扣款、退款返还 |
| `tb_points_record` | 积分流水表 | 记录积分变动（含幂等唯一索引） |
| `tb_member_level` | 会员等级配置表 | 等级权益查询 |

## 四、公共工具类

**文件位置**：`yuexiang-common/src/main/java/com/yuexiang/common/util/MoneyUtils.java`

> 金额转换是全局通用需求（优惠券模块、用户模块、订单模块都会用到），放在 common 模块中统一维护。

```java
package com.yuexiang.common.util;

import java.math.BigDecimal;
import java.math.RoundingMode;

/**
 * 金额工具类
 * <p>系统内部统一使用「分」作为金额单位，展示时转换为「元」</p>
 */
public final class MoneyUtils {

    private MoneyUtils() {}

    private static final BigDecimal HUNDRED = new BigDecimal("100");

    /**
     * 分 → 元（字符串，保留2位小数）
     *
     * @param fen 金额，单位：分
     * @return 金额字符串，单位：元，如 "99.00"
     */
    public static String fenToYuan(Long fen) {
        if (fen == null) {
            return "0.00";
        }
        return new BigDecimal(fen)
                .divide(HUNDRED, 2, RoundingMode.HALF_UP)
                .toPlainString();
    }

    /**
     * 元（字符串）→ 分
     *
     * @param yuan 金额字符串，单位：元
     * @return 金额，单位：分
     */
    public static Long yuanToFen(String yuan) {
        if (yuan == null || yuan.isEmpty()) {
            return 0L;
        }
        return new BigDecimal(yuan)
                .multiply(HUNDRED)
                .setScale(0, RoundingMode.HALF_UP)
                .longValue();
    }

    /**
     * 分 → 元（BigDecimal）
     */
    public static BigDecimal fenToYuanDecimal(Long fen) {
        if (fen == null) {
            return BigDecimal.ZERO.setScale(2);
        }
        return new BigDecimal(fen).divide(HUNDRED, 2, RoundingMode.HALF_UP);
    }
}
```

```java
package com.yuexiang.user.service.impl;

import com.yuexiang.common.exception.BusinessException;
import com.yuexiang.common.util.MoneyUtils;
import com.yuexiang.user.api.UserWalletService;
import com.yuexiang.user.domain.entity.UserWallet;
import com.yuexiang.user.domain.vo.UserWalletVO;
import com.yuexiang.user.mapper.UserWalletMapper;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.dao.DuplicateKeyException;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

/**
 * 用户钱包服务实现
 *
 * <p>并发安全策略：使用 SELECT FOR UPDATE（悲观锁）。</p>
 * <p>
 *   为什么不用"乐观锁 + 重试"？
 *   MySQL InnoDB 默认 REPEATABLE READ 隔离级别下，同一事务中的普通 SELECT
 *   始终返回事务开始时的快照（快照读），重试时读到的 version 仍是旧值，
 *   导致重试必定失败。SELECT FOR UPDATE 执行当前读，可读到最新已提交数据，
 *   且对行加排他锁，直到事务结束才释放。
 * </p>
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class UserWalletServiceImpl implements UserWalletService {

    private final UserWalletMapper walletMapper;

    @Override
    @Transactional(rollbackFor = Exception.class)
    public boolean deductBalance(Long userId, Long amount) {
        if (amount == null || amount <= 0) {
            throw new BusinessException("扣款金额必须大于0");
        }

        // 1. 悲观锁查询（FOR UPDATE），确保读到最新值并锁行
        UserWallet wallet = getOrInitWalletForUpdate(userId);

        // 2. 余额校验
        if (wallet.getBalance() < amount) {
            throw new BusinessException(
                    String.format("余额不足，当前余额：%s 元，需要扣款：%s 元",
                            MoneyUtils.fenToYuan(wallet.getBalance()),
                            MoneyUtils.fenToYuan(amount)));
        }

        // 3. 扣款（行已被 FOR UPDATE 锁定，不会有并发冲突）
        int rows = walletMapper.deductBalance(userId, amount);
        if (rows == 0) {
            // 理论上不会走到这里（FOR UPDATE 已锁行且余额已校验）
            throw new BusinessException("扣款失败，请重试");
        }

        log.info("钱包扣款成功 | userId={}, amount={}分, 原余额={}分, 新余额={}分",
                userId, amount, wallet.getBalance(), wallet.getBalance() - amount);
        return true;
    }

    @Override
    @Transactional(rollbackFor = Exception.class)
    public boolean refundBalance(Long userId, Long amount) {
        if (amount == null || amount <= 0) {
            throw new BusinessException("退款金额必须大于0");
        }

        // 1. 悲观锁查询
        UserWallet wallet = getOrInitWalletForUpdate(userId);

        // 2. 退款
        int rows = walletMapper.refundBalance(userId, amount);
        if (rows == 0) {
            throw new BusinessException("退款失败，请重试");
        }

        log.info("钱包退款成功 | userId={}, amount={}分, 原余额={}分, 新余额={}分",
                userId, amount, wallet.getBalance(), wallet.getBalance() + amount);
        return true;
    }

    @Override
    public UserWalletVO getWalletInfo(Long userId) {
        // 只读查询，不需要 FOR UPDATE
        UserWallet wallet = getOrInitWallet(userId);

        UserWalletVO vo = new UserWalletVO();
        vo.setUserId(userId);
        vo.setBalance(wallet.getBalance());
        vo.setTotalConsume(wallet.getTotalConsume());
        vo.setTotalRecharge(wallet.getTotalRecharge());
        vo.setFrozenBalance(wallet.getFrozenBalance());
        vo.setBalanceYuan(MoneyUtils.fenToYuan(wallet.getBalance()));
        vo.setTotalConsumeYuan(MoneyUtils.fenToYuan(wallet.getTotalConsume()));
        vo.setTotalRechargeYuan(MoneyUtils.fenToYuan(wallet.getTotalRecharge()));
        return vo;
    }

    @Override
    public boolean checkBalance(Long userId, Long amount) {
        UserWallet wallet = walletMapper.selectByUserId(userId);
        if (wallet == null) {
            return false;
        }
        return wallet.getBalance() >= amount;
    }

    @Override
    @Transactional(rollbackFor = Exception.class)
    public void initWallet(Long userId) {
        // 先检查是否已存在（快速路径）
        UserWallet existing = walletMapper.selectByUserId(userId);
        if (existing != null) {
            log.debug("钱包已存在，跳过初始化 | userId={}", userId);
            return;
        }

        // 插入新钱包，捕获唯一索引冲突（并发安全）
        try {
            UserWallet wallet = new UserWallet();
            wallet.setUserId(userId);
            wallet.setBalance(0L);
            wallet.setFrozenBalance(0L);
            wallet.setTotalConsume(0L);
            wallet.setTotalRecharge(0L);
            wallet.setVersion(1); // 与 DDL DEFAULT 1 保持一致
            walletMapper.insert(wallet);
            log.info("初始化用户钱包 | userId={}", userId);
        } catch (DuplicateKeyException e) {
            // 并发情况下另一个线程已创建，静默忽略
            log.debug("钱包已被并发创建，跳过 | userId={}", userId);
        }
    }

    // ==================== 私有方法 ====================

    /**
     * 查询钱包（不加锁），不存在则自动初始化
     */
    private UserWallet getOrInitWallet(Long userId) {
        UserWallet wallet = walletMapper.selectByUserId(userId);
        if (wallet == null) {
            initWallet(userId);
            wallet = walletMapper.selectByUserId(userId);
        }
        return wallet;
    }

    /**
     * 查询钱包（FOR UPDATE 加锁），不存在则自动初始化
     * <p>必须在 @Transactional 方法内调用</p>
     */
    private UserWallet getOrInitWalletForUpdate(Long userId) {
        UserWallet wallet = walletMapper.selectByUserIdForUpdate(userId);
        if (wallet == null) {
            initWallet(userId);
            wallet = walletMapper.selectByUserIdForUpdate(userId);
        }
        return wallet;
    }
}
```

### 10.2 UserPointsServiceImpl

**文件位置**：`yuexiang-module-user-biz/src/main/java/com/yuexiang/user/service/impl/UserPointsServiceImpl.java`

```java
package com.yuexiang.user.service.impl;

import com.yuexiang.common.exception.BusinessException;
import com.yuexiang.user.api.UserPointsService;
import com.yuexiang.user.constants.UserConstants;
import com.yuexiang.user.domain.entity.MemberLevel;
import com.yuexiang.user.domain.entity.PointsRecord;
import com.yuexiang.user.domain.entity.User;
import com.yuexiang.user.domain.entity.UserInfo;
import com.yuexiang.user.domain.vo.UserInfoVO;
import com.yuexiang.user.mapper.MemberLevelMapper;
import com.yuexiang.user.mapper.PointsRecordMapper;
import com.yuexiang.user.mapper.UserInfoMapper;
import com.yuexiang.user.mapper.UserMapper;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.dao.DuplicateKeyException;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.math.BigDecimal;

/**
 * 用户积分服务实现
 *
 * <p>幂等策略（双重保障）：</p>
 * <ol>
 *   <li>Service 层先 SELECT 查询 tb_points_record 是否已存在相同业务记录（快速路径）</li>
 *   <li>INSERT 时依赖 uk_points_biz(user_id, biz_type, biz_id) 唯一索引，
 *       捕获 DuplicateKeyException 作为并发兜底</li>
 * </ol>
 *
 * <p>积分扣减策略：使用 SELECT FOR UPDATE 锁定当前积分，精确计算可扣减数量，
 *    积分不足时扣减到 0，不报错</p>
 */
@Slf4j
@Service
@RequiredArgsConstructor
public class UserPointsServiceImpl implements UserPointsService {

    private final UserMapper userMapper;
    private final UserInfoMapper userInfoMapper;
    private final PointsRecordMapper pointsRecordMapper;
    private final MemberLevelMapper memberLevelMapper;

    @Override
    @Transactional(rollbackFor = Exception.class)
    public boolean addPoints(Long userId, Integer points, String bizType, Long bizId, String description) {
        if (points == null || points <= 0) {
            throw new BusinessException("增加积分数必须大于0");
        }

        // ===== 1. 幂等校验（快速路径） =====
        PointsRecord existing = pointsRecordMapper.selectByBiz(userId, bizType, bizId);
        if (existing != null) {
            log.warn("积分已赠送过，幂等跳过 | userId={}, bizType={}, bizId={}", userId, bizType, bizId);
            return true;
        }

        // ===== 2. 增加用户积分 =====
        // UPDATE 语句自带行锁，并发安全
        int rows = userInfoMapper.addPoints(userId, points);
        if (rows == 0) {
            throw new BusinessException("用户不存在，userId=" + userId);
        }

        // ===== 3. 查询变动后的积分余额 =====
        int currentPoints = getUserPoints(userId);

        // ===== 4. 记录积分流水（唯一索引兜底幂等） =====
        PointsRecord record = new PointsRecord();
        record.setUserId(userId);
        record.setType(mapBizTypeToRecordType(bizType, true));
        record.setPoints(points);           // 正数
        record.setBalance(currentPoints);
        record.setBizType(bizType);
        record.setBizId(bizId);
        record.setDescription(description);

        try {
            pointsRecordMapper.insert(record);
        } catch (DuplicateKeyException e) {
            // 并发场景：另一个线程已先插入，本次事务回滚（addPoints 也回滚）
            log.warn("积分幂等拦截(唯一索引) | userId={}, bizType={}, bizId={}", userId, bizType, bizId);
            throw e; // 抛出让 @Transactional 回滚，避免积分重复增加
        }

        log.info("积分增加 | userId={}, +{}积分, 余额={}积分, bizType={}, bizId={}",
                userId, points, currentPoints, bizType, bizId);

        // ===== 5. 检查等级变更 =====
        checkAndUpdateLevel(userId);

        return true;
    }

    @Override
    @Transactional(rollbackFor = Exception.class)
    public boolean deductPoints(Long userId, Integer points, String bizType, Long bizId, String description) {
        if (points == null || points <= 0) {
            throw new BusinessException("扣减积分数必须大于0");
        }

        // ===== 1. 幂等校验（快速路径） =====
        PointsRecord existing = pointsRecordMapper.selectByBiz(userId, bizType, bizId);
        if (existing != null) {
            log.warn("积分已扣减过，幂等跳过 | userId={}, bizType={}, bizId={}", userId, bizType, bizId);
            return true;
        }

        // ===== 2. 悲观锁读取当前积分（FOR UPDATE，当前读） =====
        Integer currentPoints = userInfoMapper.selectPointsByUserIdForUpdate(userId);
        if (currentPoints == null) {
            throw new BusinessException("用户不存在，userId=" + userId);
        }

        // ===== 3. 计算实际扣减数量（不足则扣到0） =====
        int actualDeducted;
        if (currentPoints <= 0) {
            log.warn("用户积分为0，跳过扣减 | userId={}", userId);
            actualDeducted = 0;
        } else {
            actualDeducted = Math.min(currentPoints, points);
            int newPoints = currentPoints - actualDeducted;
            userInfoMapper.setPoints(userId, newPoints);
        }

        // ===== 4. 记录积分流水（只有实际扣减了才记录） =====
        if (actualDeducted > 0) {
            int balanceAfter = currentPoints - actualDeducted;

            PointsRecord record = new PointsRecord();
            record.setUserId(userId);
            record.setType(mapBizTypeToRecordType(bizType, false));
            record.setPoints(-actualDeducted);   // 负数
            record.setBalance(balanceAfter);
            record.setBizType(bizType);
            record.setBizId(bizId);
            record.setDescription(description
                    + (actualDeducted < points
                    ? String.format("（原应扣%d，实际扣%d）", points, actualDeducted) : ""));

            try {
                pointsRecordMapper.insert(record);
            } catch (DuplicateKeyException e) {
                log.warn("积分扣减幂等拦截(唯一索引) | userId={}, bizType={}, bizId={}",
                        userId, bizType, bizId);
                throw e; // 回滚
            }
        }

        log.info("积分扣减 | userId={}, 应扣={}, 实扣={}, 原积分={}, 余额={}, bizType={}, bizId={}",
                userId, points, actualDeducted, currentPoints,
                currentPoints - actualDeducted, bizType, bizId);

        // ===== 5. 检查等级变更（可能降级） =====
        checkAndUpdateLevel(userId);

        return true;
    }

    @Override
    @Transactional(rollbackFor = Exception.class)
    public boolean checkAndUpdateLevel(Long userId) {
        // 1. 获取当前积分
        int points = getUserPoints(userId);

        // 2. 获取当前等级
        Integer currentLevel = userInfoMapper.selectMemberLevel(userId);
        if (currentLevel == null) {
            currentLevel = 1;
        }

        // 3. 根据积分计算应有等级
        MemberLevel targetLevel = memberLevelMapper.selectByPoints(points);
        if (targetLevel == null) {
            log.warn("未找到匹配的会员等级 | userId={}, points={}", userId, points);
            return false;
        }

        // 4. 等级发生变化才更新（支持升级和降级，使用 != 判断）
        if (!currentLevel.equals(targetLevel.getLevel())) {
            String direction = targetLevel.getLevel() > currentLevel ? "升级" : "降级";
            userInfoMapper.updateMemberLevel(userId, targetLevel.getLevel());
            log.info("会员等级{} | userId={}, points={}, {} → {} ({})",
                    direction, userId, points,
                    currentLevel, targetLevel.getLevel(), targetLevel.getName());
            return true;
        }

        return false;
    }

    @Override
    public int getUserPoints(Long userId) {
        Integer points = userInfoMapper.selectPointsByUserId(userId);
        return points != null ? points : 0;
    }

    @Override
    public UserInfoVO getUserInfo(Long userId) {
        // 1. 查询用户详情（积分、等级）
        UserInfo userInfo = userInfoMapper.selectByUserId(userId);
        if (userInfo == null) {
            return null;
        }

        // 2. 查询用户基本信息（昵称在 tb_user 表中）
        User user = userMapper.selectById(userId);

        int points = userInfo.getPoints() != null ? userInfo.getPoints() : 0;
        int level = userInfo.getLevel() != null ? userInfo.getLevel() : 1;

        // 3. 查询当前等级信息
        MemberLevel currentMemberLevel = memberLevelMapper.selectByPoints(points);

        // 4. 查询下一等级
        MemberLevel nextLevel = memberLevelMapper.selectNextLevel(level);

        // 5. 组装 VO
        UserInfoVO vo = new UserInfoVO();
        vo.setUserId(userId);
        vo.setNickname(user != null ? user.getNickName() : null);
        vo.setPoints(points);
        vo.setMemberLevel(level);
        vo.setMemberLevelName(currentMemberLevel != null ? currentMemberLevel.getName() : "青铜");
        vo.setDiscount(currentMemberLevel != null
                ? currentMemberLevel.getDiscount()
                : new BigDecimal("1.00"));

        // 6. 计算距离下一等级还需积分
        if (nextLevel != null) {
            vo.setPointsToNextLevel(Math.max(0, nextLevel.getMinPoints() - points));
        } else {
            vo.setPointsToNextLevel(0); // 已是最高等级
        }

        return vo;
    }

    // ==================== 私有方法 ====================

    /**
     * 将业务类型映射为积分记录类型
     *
     * @param bizType 业务类型
     * @param isAdd   true=增加场景，false=扣减场景
     * @return 记录类型
     */
    private int mapBizTypeToRecordType(String bizType, boolean isAdd) {
        if (bizType == null) {
            return isAdd ? UserConstants.POINTS_TYPE_CONSUME : UserConstants.POINTS_TYPE_REFUND;
        }
        switch (bizType) {
            case UserConstants.BIZ_VOUCHER_BUY:
                return UserConstants.POINTS_TYPE_CONSUME;    // 2
            case UserConstants.BIZ_VOUCHER_REFUND:
                return UserConstants.POINTS_TYPE_REFUND;     // 6
            case UserConstants.BIZ_SIGN_IN:
                return UserConstants.POINTS_TYPE_SIGN_IN;    // 1
            case UserConstants.BIZ_ACTIVITY:
                return UserConstants.POINTS_TYPE_ACTIVITY;   // 3
            case UserConstants.BIZ_EXCHANGE:
                return UserConstants.POINTS_TYPE_EXCHANGE;   // 5
            default:
                return isAdd
                        ? UserConstants.POINTS_TYPE_CONSUME  // 2
                        : UserConstants.POINTS_TYPE_REFUND;  // 6
        }
    }
}
```

```java
package com.yuexiang.user.service;

import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.baomidou.mybatisplus.extension.plugins.pagination.Page;
import com.yuexiang.user.domain.entity.PointsRecord;

@Service

    /**
     * 分页查询用户积分流水
     *

     * @param type   变动类型（可选过滤）
     * @param page   页码
     * @param size   每页条数
     * @return 分页结果
     */
    public Page<PointsRecord> queryRecords(Long userId, Integer type, int page, int size) {
        LambdaQueryWrapper<PointsRecord> wrapper = new LambdaQueryWrapper<PointsRecord>()
                .eq(PointsRecord::getUserId, userId)
                .eq(type != null, PointsRecord::getType, type)
                .orderByDesc(PointsRecord::getCreateTime);

        return pointsRecordMapper.selectPage(new Page<>(page, size), wrapper);
    }
}
```

### 12.2 VoucherOrderServiceImpl 集成代码

**文件位置**：`yuexiang-module-voucher-biz/src/main/java/.../service/impl/VoucherOrderServiceImpl.java`

```java
package com.yuexiang.voucher.service.impl;

import com.yuexiang.common.exception.BusinessException;
import com.yuexiang.user.api.UserWalletService;
import com.yuexiang.user.api.UserPointsService;
import com.yuexiang.user.constants.UserConstants;
import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.stereotype.Service;
import org.springframework.transaction.annotation.Transactional;

import java.time.LocalDateTime;

@Slf4j
@Service
@RequiredArgsConstructor
public class VoucherOrderServiceImpl implements VoucherOrderService {

    // 来自 user-api 的接口（运行时由 Spring 注入 user-biz 中的实现）
    private final UserWalletService userWalletService;
    private final UserPointsService userPointsService;

    // ... 其他已有依赖（orderMapper, voucherMapper 等）...

    /**
     * ============================================
     *         支付订单
     * ============================================
     * 事务范围：订单状态更新 + 钱包扣款 + 积分赠送
     */
    @Override
    @Transactional(rollbackFor = Exception.class)
    public boolean payOrder(Long orderId, Long userId, Integer payType) {

        // ---- 1. 查询并校验订单 ----
        VoucherOrder order = orderMapper.selectById(orderId);
        if (order == null || order.getStatus() != OrderStatus.UNPAID) {
            throw new BusinessException("订单不存在或状态异常");
        }
        Voucher voucher = voucherMapper.selectById(order.getVoucherId());

        // 实付金额（单位：分）
        Long payAmount = voucher.getPayValue();

        // ---- 2. 根据支付方式处理 ----
        switch (payType) {
            case UserConstants.PAY_TYPE_BALANCE:
                // 余额支付：先检查再扣款
                if (!userWalletService.checkBalance(userId, payAmount)) {
                    throw new BusinessException("余额不足");
                }
                userWalletService.deductBalance(userId, payAmount);
                break;
            case UserConstants.PAY_TYPE_WECHAT:
            case UserConstants.PAY_TYPE_ALIPAY:
                // TODO: 接入微信/支付宝支付
                throw new BusinessException("暂不支持该支付方式，请使用余额支付");
            default:
                throw new BusinessException("不支持的支付方式：" + payType);
        }

        // ---- 3. 更新订单状态为已支付 ----
        order.setStatus(OrderStatus.PAID);
        order.setPayType(payType);
        order.setPayTime(LocalDateTime.now());
        orderMapper.updateById(order);

        // ---- 4. 赠送积分 ----
        int earnedPoints = UserConstants.calculatePoints(payAmount);
        if (earnedPoints > 0) {
            userPointsService.addPoints(
                    userId,
                    earnedPoints,
                    UserConstants.BIZ_VOUCHER_BUY,
                    orderId,
                    "购买优惠券[" + voucher.getTitle() + "]赠送积分"
            );
        }

        log.info("订单支付完成 | orderId={}, userId={}, payType={}, payAmount={}分, earnedPoints={}",
                orderId, userId, payType, payAmount, earnedPoints);

        return true;
    }

    /**
     * ============================================
     *         退款
     * ============================================
     * 事务范围：订单状态更新 + 钱包退款 + 积分扣回
     */
    @Override
    @Transactional(rollbackFor = Exception.class)
    public boolean refundOrder(Long orderId, Long userId) {

        // ---- 1. 查询并校验订单 ----
        VoucherOrder order = orderMapper.selectById(orderId);
        if (order == null) {
            throw new BusinessException("订单不存在");
        }
        if (order.getStatus() == OrderStatus.REFUNDED) {
            log.warn("订单已退款，跳过 | orderId={}", orderId);
            return true;
        }
        if (order.getStatus() != OrderStatus.UNUSED) {
            throw new BusinessException("当前订单状态不可退款");
        }

        Voucher voucher = voucherMapper.selectById(order.getVoucherId());
        Long refundAmount = voucher.getPayValue();

        // ---- 2. 更新订单状态为已退款 ----
        order.setStatus(OrderStatus.REFUNDED);
        order.setRefundTime(LocalDateTime.now());
        orderMapper.updateById(order);

        // ---- 3. 根据原支付方式返还 ----
        switch (order.getPayType()) {
            case UserConstants.PAY_TYPE_BALANCE:
                // 余额支付 → 返还余额
                userWalletService.refundBalance(userId, refundAmount);
                break;
            case UserConstants.PAY_TYPE_WECHAT:
            case UserConstants.PAY_TYPE_ALIPAY:
                // TODO: 调用三方退款接口
                log.warn("三方支付退款暂未实现，需人工处理 | orderId={}, payType={}, amount={}分",
                        orderId, order.getPayType(), refundAmount);
                break;
            default:
                log.error("未知支付方式，无法退款 | orderId={}, payType={}", orderId, order.getPayType());
        }

        // ---- 4. 扣回赠送的积分 ----
        int refundPoints = UserConstants.calculatePoints(refundAmount);
        if (refundPoints > 0) {
            userPointsService.deductPoints(
                    userId,
                    refundPoints,
                    UserConstants.BIZ_VOUCHER_REFUND,
                    orderId,
                    "优惠券[" + voucher.getTitle() + "]退款扣回积分"
            );
        }

        log.info("订单退款完成 | orderId={}, userId={}, payType={}, refund={}分, deductPoints={}",
                orderId, userId, order.getPayType(), refundAmount, refundPoints);

        return true;
    }
}
```

## 十五、注意事项

### 15.1 并发安全
- 钱包扣款/退款使用 **SELECT FOR UPDATE 悲观锁**，而非乐观锁重试
- 原因：MySQL InnoDB 默认 REPEATABLE READ 隔离级别下，同一事务内的普通 SELECT 为快照读，重试时读不到最新 version，导致重试必定失败
- `SELECT ... FOR UPDATE` 执行当前读，加排他行锁直到事务提交
- version 字段保留并自增，仅用于审计追踪，不用于并发控制
- 实体类 **不加 `@Version` 注解**，避免 MyBatis Plus 自动管理与手写 SQL 冲突

### 15.2 幂等性（双重保障）
- **第一层（快速路径）**：Service 层先 SELECT 查询 `tb_points_record` 是否已存在相同 `(user_id, biz_type, biz_id)` 记录
- **第二层（并发兜底）**：DDL 中 `uk_points_biz(user_id, biz_type, biz_id)` **唯一索引**，INSERT 时捕获 `DuplicateKeyException` 并让事务回滚
- 幂等命中时返回 `true`（成功），不重复执行业务

### 15.3 事务一致性
- 当前采用 **同步调用 + 同一事务** 方案
- 订单状态更新、钱包扣款、积分赠送在同一个 `@Transactional` 中
- 任一步骤失败，整体回滚

### 15.4 积分规则
- 计算公式：`积分 = 实付金额（分）/ 100`
- 不足 1 元（100 分）的部分不计积分
- 所有金额单位统一为 **分**

### 15.5 等级变更
- 支持 **升级和降级**（使用 `!=` 判断，非 `>` 判断）
- 积分增加后可能升级，积分减少后可能降级
- 等级配置从 `tb_member_level` 表读取

### 15.6 退款积分不足
- 退款时用户可能已使用过积分
- 处理策略：积分不足时 **扣到 0**，不报错
- 使用 FOR UPDATE 锁定后精确计算可扣减数量，避免快照读问题
- 流水记录中标注实际扣减数与应扣减数的差异

### 15.7 钱包自动初始化
- 查询钱包时若不存在则 **自动创建**（余额为 0）
- 使用 `try-catch DuplicateKeyException` 处理并发初始化
- 用户注册流程中也应调用 `initWallet()`
- version 初始值为 **1**，与 DDL DEFAULT 保持一致

### 15.8 实体字段与 DDL 对齐
- `UserWallet.frozenBalance` 对应 DDL 列 `frozen_balance`
- `UserInfo.level` 对应 DDL 列 `level`（加 `@TableField("level")` 显式标注）
- `MemberLevel.discount` 类型为 `BigDecimal`，对应 DDL `DECIMAL(3,2)`
- `PointsRecord.type` 的枚举值与 DDL 注释保持一致

### 15.9 跨表查询
- 用户昵称（`nick_name`）在 `tb_user` 表中，不在 `tb_user_info` 表中
- `getUserInfo()` 方法需要联合查询 `UserMapper` 和 `UserInfoMapper`

### 15.10 三方支付退款
- 当前仅实现余额支付退款
- 微信/支付宝退款在 `refundOrder` 中预留了 switch 分支和日志
- 需后续接入三方退款 API 时补充实现
