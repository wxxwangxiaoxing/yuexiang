# 悦享生活项目配置说明

## 环境变量配置

为了保护敏感信息，项目使用环境变量进行配置。请在部署时设置以下环境变量：

### 数据库配置

```bash
MYSQL_URL=jdbc:mysql://your-host:3306/yuexiang?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
MYSQL_USERNAME=your-username
MYSQL_PASSWORD=your-password
```

### Redis配置

```bash
REDIS_HOST=your-redis-host
REDIS_PORT=6379
REDIS_DATABASE=0
REDIS_PASSWORD=your-redis-password
```

### AI服务配置

```bash
# OpenAI配置
OPENAI_API_KEY=your-openai-api-key
OPENAI_BASE_URL=https://api.openai.com

# DeepSeek配置
DEEPSEEK_API_KEY=your-deepseek-api-key

# Qwen配置
QWEN_API_KEY=your-qwen-api-key
```

## 业务配置

项目的业务配置项都在`application.yml`中的`yuexiang.business`节点下：

### 积分配置

```yaml
yuexiang:
  business:
    points:
      default-level: 1          # 默认用户等级
      max-points: 1000000       # 最大积分值
      min-points: 0             # 最小积分值
      bonus-points: 30          # 奖励积分
      bonus-trigger-days: 7     # 奖励触发天数
```

### 签到配置

```yaml
    sign:
      repair-cost-points: 50    # 补签消耗积分
      repair-max-per-month: 3   # 每月最大补签次数
      repair-window-days: 7     # 补签窗口天数
```

### 钱包配置

```yaml
    wallet:
      max-retry: 3              # 最大重试次数
      max-amount: 100000000     # 最大金额（分）
```

### 账户配置

```yaml
    account:
      max-err-l1: 3             # 一级错误次数限制
      max-err-l2: 5             # 二级错误次数限制
      max-err-l3: 10            # 三级错误次数限制
      lock-minutes-l1: 120      # 一级锁定时间（分钟）
      lock-minutes-l2: 1440     # 二级锁定时间（分钟）
      cooling-days: 7           # 冷却天数
      bcrypt-strength: 10       # BCrypt加密强度
```

### 博客配置

```yaml
    blog:
      max-month-lookback: 13    # 最大回溯月数
```

## 配置优先级

配置的优先级从高到低：

1. 环境变量
2. 命令行参数
3. application-{profile}.yml
4. application.yml

## 开发环境配置

在开发环境中，可以创建`application-dev.yml`文件，覆盖默认配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/yuexiang_dev
    username: root
    password: 123456

logging:
  level:
    com.yuexiang: debug
```

## 生产环境配置

在生产环境中，建议：

1. 使用环境变量管理所有敏感信息
2. 使用配置中心（如Spring Cloud Config、Nacos）管理配置
3. 定期更新密码和密钥
4. 使用加密配置（如Jasypt）

## 配置验证

项目启动时会自动验证配置项的有效性。如果配置项缺失或无效，应用将启动失败并提示错误信息。

## 配置更新

修改配置后，需要重启应用才能生效。如果需要动态更新配置，可以考虑：

1. 使用Spring Cloud Config
2. 使用Nacos配置中心
3. 使用Spring Boot Actuator的refresh端点

## 安全建议

1. **不要将敏感信息提交到代码仓库**
2. 使用环境变量或配置中心管理敏感信息
3. 定期更新密码和密钥
4. 使用加密配置存储敏感信息
5. 限制配置文件的访问权限
